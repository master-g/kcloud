# Rust 版 Claude Code 最小实现 - 完整实施计划

## 项目概述

**项目名称**: kloud  
**项目目标**: 通过 Rust 实现一个最小化的 Claude Code，学习理解 Agent 的工作原理  
**项目性质**: 学习项目

---

## 技术栈

| 组件 | 选择 |
|------|------|
| CLI 框架 | clap |
| 异步运行时 | tokio |
| HTTP 客户端 | reqwest |
| TUI | ratatui |
| 错误处理 | thiserror + anyhow |
| 日志 | tracing |
| 配置 | 环境变量 + TOML 配置文件 |
| 测试 | cargo test |
| LLM 后端 | OpenAI 兼容 API |

---

## 阶段划分

| 阶段 | 名称 | 目标 |
|------|------|------|
| 1 | 工具与执行 | Agent 循环，工具，安全机制，格式化显示 |
| 2 | 规划与协调 | TODO Write，子 Agent，技能系统，任务状态管理 |
| 3 | 内存管理 | 三层上下文压缩 |
| 4 | 并发 | 任务队列和并发执行 |
| 5 | 协作 | Agent 团队，团队协议，自主 Agent，Git worktree 隔离 |

---

## 实施策略

### 并行执行设计

本项目采用分阶段顺序实施，每个阶段内部可并行实现：

- **阶段 1**: 基础架构先行，工具模块可并行开发
- **阶段 2-5**: 依次顺序实施，每个阶段依赖前一阶段的成果

### 验证策略

每个阶段完成后通过以下方式验证：
1. 单元测试 (cargo test)
2. 集成测试 (手动测试核心功能)
3. 代码审查 (确保符合 Rust 最佳实践)

---

## 统一类型定义

所有 Agent 类型在 `src/agents/types.rs` 中统一定义：

| AgentType | 名称 | 阶段 |
|-----------|------|------|
| Main | 主 Agent | 1 |
| Coder | 代码编写 | 2, 5 |
| Reviewer | 代码审查 | 2, 5 |
| Planner | 任务规划 | 2, 5 |
| Explorer | 探索/搜索 | 2, 5 |
| Joker | 创意/娱乐 | 2, 5 |

**原则**：阶段5应复用阶段2实现的 Agent 代码，仅添加团队协作相关功能。
